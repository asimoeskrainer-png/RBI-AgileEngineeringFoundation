# Hands-On: Risk-Based Test Prioritization with MoSCoW

## Objective

As a test engineer, your task is to apply risk-based testing principles
to prioritize which user stories should be tested in the upcoming
sprint.

---

# Background

Six user stories are planned for release.

You are expected to evaluate them based on risk exposure by
considering:
- Impact
  - How severe a failure would be
- Likelihood
  - How probable the failure is to occur

Follow the guiding principle:

> "No risk, no test."

---

# Task Description

Use the MoSCoW prioritization method to decide which stories belong to
each category:

- MUST
  - Critical to test this sprint
  - Select 2 stories

- SHOULD
  - Important but not critical
  - Select 1 story

- COULD
  - Nice to have if time allows
  - Select 1 story

- WON’T
  - Out of testing scope for this sprint
  - Select 2 stories

---

# Expected Outcome

For each user story:
- Assess the risk level
- Assign a MoSCoW priority
- Document your reasoning

Explain:
- Why the story received its priority
- Which risks influenced the decision
- Which business or technical impact was considered

---

# Recommended Approach

Consider:
- Business impact
- Technical complexity
- Integration dependencies
- Security relevance
- Customer visibility
- Regression potential
- Production risk
